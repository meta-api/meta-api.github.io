Configure Connectors' authentication

Overview‚Äã

There are several ways to identify with an API:

  • Basic authentication, you don't need to authenticate yourself.
  • Authentication via a token, you need a key provided by the API and integrated in the connector configurations.
  • OAuth's authentication, a stricter configuration is required by the API to provide several parameters.

One Click Authentication‚Äã

For some APIs, we provide a one click authentication.

If available, you'll see a One Click button inside the connector

You just have to click on it to start an automatic OAuth 2 authentication and your profile will be automatically added to your connector.

Basic authentication‚Äã

You don't have to give any parameters, you can use the API without any problem !

Token Authentication‚Äã

For Token authentication you need a key provided by the API provider, here is an example where we get a key for the TMDB API. After retrieving the API key, we create a new authentication account in which we integrate the API key of the connector.

OAuth Authentication‚Äã

OAuth's authentication is different because it requires more parameters to be filled in, but its configuration is similar to that of Token authentication. An example is available in the Slack API integration. As for the authentication by token one creates a new account of authentication and one fills the fields necessary to the configuration of the account.

Advance configuration for OAuth2 Refresh‚Äã

You can add this custom object inside a OAuth2 flows object to add settings : 

"X-Refresh-Settings": {
    "Content-Type": "application/x-www-form-urlencoded",
    "defaultExpireSec": 120
}
  • Some APIs only support a form as a Content-Type
  • You can force the expiration delay of a token with defaultExpireSec setting

Here is a full example with Salesforce: 

"securitySchemes": {
    "oauth2": {
    "type": "oauth2",
    "flows": {
        "authorizationCode": {
        "authorizationUrl": "https://server.my.salesforce.com/services/oauth2/authorize",
        "tokenUrl": "https://server.my.salesforce.com/services/oauth2/token",
        "scopes": {
            "offline_access": "To get a refresh token",
            "full": "Full Access to any APIs",
            "openid": "Allows access to the current, logged in user‚Äôs unique identifier for OpenID Connect apps. In the OAuth 2.0 user-agent flow and the OAuth 2.0 web server flow, use the openid scope. In addition to the access token, this scope enables you to receive a signed ID token that conforms to the OpenID Connect specifications.",
            "lightning": "Allows hybrid apps to directly obtain Lightning child sessions through the OAuth 2.0 hybrid app token flow and hybrid app refresh token flow.",
            "content": "Access content ressources",
            "api": "Manage user data via API"
        },
        "refreshUrl": "https://server.my.salesforce.com/services/oauth2/token",
        "X-Refresh-Settings": {
            "Content-Type": "application/x-www-form-urlencoded",
            "defaultExpireSec": 120
        }
        }
    }
    }
}

Known settings on provider‚Äã

Google APIs:

You must add two additional parameters on Google's API for enable the refresh_token and force changes if you want to add your own profile : 

access_type: offline
prompt: consent

Salesforce APIs:

Salesforce's Identity Provider don't provide an expiration time on their OAuth tokens. So you can force the delay with this parameters inside the flows OAuth's config. You must also force the use of a form over a JSON object.

"X-Refresh-Settings": {
    "Content-Type": "application/x-www-form-urlencoded",
    "defaultExpireSec": 120
}

You'll find a full example above.
