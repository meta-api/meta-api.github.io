Quick Start (10 min)

Video#

You can have a look to this presentation video (french audio with english subtitles) with show how to create a spell :

Guide#

Create or access a new Spell#

You can access the example Spell or the new spell directly from the Spell menu :

The anatomy of a spell#

A spell have 2 importants parts :

  • On the left, a column with all the tools you'll need to build your Spell : adding and editing a connector, testing it and deploying it.
  • On the right : a code editor to write the JS and TS code to "glue" all the connectors and transform the data the way you want.

Testing a spell#

To test your Spell, click on the button to push your pending changes to the server, go to the tab and click .

And will trigger a request with the given parameters provided on test request panel and you'll the response inside the response panel.

Adding a connector#

To add a new connector inside your Spell and generate the right code for the code editor

  1. Put the cursor into the right position in your code

The code will be added to the current cursor position. You'll be able to move that code later and to generate it from the "Warning" sign inside a connector.

  1. Go to tab

You will be able here to add a new connector with this button :

Inside the popup, search the right connector for you and then select the desired endpoint.

You'll see then the detail of parameters and the response's schema.

  1. Configure your new connector

First, configure an authentication account if the API is secured. Click on

Then, you can fill all parameters with values or connect a variable from your code by clicking on the icon.

All the changes will be automatically saved.

Don't forget to save your spell to propagate all the changes in order to test it.

Add dependencies#

The tab will allow you to search and add libraries to your code.

Search NPM packages with the search toolbar.

You will then access the lib using the alias defined on each package.

Don't forget to save to install the packages inside your code.

Deploy your Spell#

Go to tab to create a new version of your code with button.

Then, click on button to push this version into production.

All the details (like URL) for using this freshly created Spell into your apps will be available under the section on top of this panel.
