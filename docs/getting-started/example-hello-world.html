Example: A gif from a movie

In this part, we'll guide you to build a Spell. The objectives of it will be to retrieve a popular movie from TMDB then to get a gif from Giphy associated from the title and send it to Slack channel.

Create the Spell​

First of all, we will create a new Spell. Go to your list of Spells and click on the Create a new Spell button and fill out the form that appears like so for the name:

A gif from a movie

And for the description:

This Spell take the most popular movie from TMDB and fetch a gif from Giphy. It then sends it to a Slack channel with the gif and information of the movie.

Then click Create and you will redirected to the Spell's editor interface. It should look like this:

Create the Spell workflow​

As said in the introduction, we will use TMDB to retrieve a popular movie, then Giphy to search a gif related to it and will send to a Slack channel.

Add TMDB​

Click on the Add a Connector button, a new modal appears and show you the API catalog of Meta-API that you have access.

Search for TMDB in the input field and click on the card that is named TMDB.
Here, you'll have all the Connectors (API endpoints) available for this API. Each Connector is grouped by the URL (ex: /movie/popular) and you will see the GET, POST, PUT/PATCH and DELETE action if available.

Now search for movie popular and click on the Get Popular, the URL is: /movie/popular.
The third panel shows you a quick summary of the endpoint: if you need an authentication; the differents parameters; the response sample; and the response schema.

Click on Add a Connector and now the Connector for popular movie on TMDB is now linked to the Spell.

Copy the code and paste it on the code editor on the right panel (you can replace all of the code previously generate) and add a return statement: return tmdbResult.
Now the editor should look like this:

/* Start of TMDB connector */
const tmdbResult: Datagetmoviepopular = await connectorService.config("61546cea61dd0e1ae9f621a7", {});
/* End of TMDB connector */

return tmdbResult;
note

The id 6064463e0524b4ef5fc98f38 is generate automaticaly and won't be the same for you. This identifier is important and should correspond to your Connector's id, otherwise the Spell can execute the request.
You can retrieve the id on the Connector detail's panel on the left by click on it.

Configure TMDB authentication​

The connector is now ready to be configure. In the Used Connectors panel, you can see that the Connector is highlight in red, showing an alert : The authentication accountis not configured on this Connector. That means we need to configure it up.

To configure it, click on the TMBD Connector and open its details. You'll see, in the settings tab the part on the authentication saying Authentication not setted yet. By clicking on the dropdown, you'll find several actions.
Select "Create a new authentication account", a modal shows up telling you to give a Label to the auth account and an ApiKey.

Add a label (ex: "TMDB api key") and let's retrieve your ApiKey.

Retrieve TMDB key​

First you'll have to register to TMDB, go to the TMDB signup page and fill the form. Then go to your settings, on the API tab. The API Key (v3 auth) is the one we are looking for (see image below)

Copy the key then return to Meta-API and paste the key into the ApiKey field (see image below) and click "Save and quit"

The authentication account is by default selectionned to the Connector's auth and we're ready to run some test.

Run the Spell by clicking the green "Run" button. Your dev environment is launching and your Spell is being prepared. Then it will launch a request.
The tab "Test" opens and show you the results in the "Response" insert. It should looks like this:

We're done with TMDB for now, let's see how is it done for Giphy now.

Add GIPHY​

Adding a Connector is always the same:

  1. Click "Add a Connector"
  2. Search for and API and click on the right card
  3. Looks for a Connector
  4. Add it to the Spell

For Giphy, we will have to search the API Giphy then the GET Connector Translate Phrase to Gifs (URL: /gifs/translate) and after read the description of the Connector, click on "Add a Connector".

Paste the code in the code editor, between the *TMDB Connector and the return statement*. you should have something as follow:

/* Start of TMDB connector */
const tmdbResult: Datagetmoviepopular = await connectorService.config("60644d453750c0660ceb5275", {});
/* End of TMDB connector */

/* Start of Giphy connector */
const giphyResult: Datagetgifstranslate = await connectorService.config("60649d5c0524b402e8c9904d", {});
/* End of Giphy connector */

return tmdbResult;

Configure GIPHY authentication​

As TMDB the Giphy Connector needs an authentication. Go to the signup page of Giphy and create an account. Then head at the developers programs dashboard.
You'll need to create an app in order to get an API key:

  • Click "Create an App"
  • Select "Select API" then "Next step"
  • Add an App Name (ex: "Meta-API Test") and a description (ex: "This app is for the example of Meta-API "A gif from a movie".")
  • Click "Create App"

You have now a Giphy API key, to use in Meta-API, follow the same instruction as for TMDB:

  1. Click on the Connector
  2. Select "Create a new authentication account"
  3. Give a label and paste the API key

Linked TMDB and Giphy: retrieve a Gif from the movie's name​

If you look on the left, the Connector for Giphy has an alert who mention a missing a mandatory parameter: s. If you open the Connector's details you will see several parameters.
In order to link TMDB and Giphy and have a gif associated, we will have to link the title of the first movie returned and the query params s.
We are going to add a new variable in our code to only get the title. The code will be between the TMDB Connector and the Giphy Connector as follow:

/* Start of TMDB connector */
const tmdbResult: Datagetmoviepopular = await connectorService.config("60644d453750c0660ceb5275", {});
/* End of TMDB connector */

const { original_title } = tmdbResult.results[0];

/* Start of Giphy connector */
const giphyResult: Datagetgifstranslate = await connectorService.config("60649d5c0524b402e8c9904d", {});
/* End of Giphy connector */

return tmdbResult;

We can now linked the variable original_title to the params s. In the Giphy Connector, you can see on the left of the input of s a button with a chain. Click on it and a new block will appears that will allow you to select any variable from your code (see image below). Select original_title and it will link automaticaly.
After the selection is done, you can see the field is now read-only and has changed: this shows you the linked variable.

Let's change our return statement to see the Giphy result, especially the data object, like this:

return giphyResult.data;

Now, we can Run our Spell and see what we'll get:

Add Slack​

Same as TMDB and Giphy, we're looking for the Slack API and for the Connector /chat.postMessage

note

Some Connector doesn't have a name but we can still find them through their URL. Also, the search engine looks in the description too.

Copy the code into the editor and we will be able to set up the OAuth for Slack.

So far you should have this in the code editor:

/* Start of TMDB connector */
const tmdbResult: Datagetmoviepopular = await connectorService.config("60644d453750c0660ceb5275", {});
/* End of TMDB connector */

const { original_title } = tmdbResult.results[0];

/* Start of Giphy connector */
const giphyResult: Datagetgifstranslate = await connectorService.config("60649d5c0524b402e8c9904d", {});
/* End of Giphy connector */

/* Start of Slack connector */
const slackResult: Datapostchatpostmessage = await connectorService.config("6065847ca27fa1c947ff016d", {});
/* End of Slack connector */

return giphyResult.data;

Configure Slack OAuth authentication​

Slack is sightly different than other authentication: we will use OAuth.
Open the Slack Connector's detail panel, click on the dropdown under Authentication then select "Create a new authentication account".
You'll notice a different modal, you'll have to configure the authentication within Meta-API and Slack. follow this step-by-step to understand how to set up the OAuth:

  1. Add a Label to the Slack authentication (ex: Slack auth)

  2. Retrieve redirects URL for OAuth

As OAuth will contact a server to made the authentication, we provide you a unique URL to this auth.
By clicking "Click here to get your personalized link" button, you will be prompt a URL like this, where client_id and connector_id are your own id:

https://api.meta-api.io/api/oauth/connect/{client_id}-{connector_id}/callback

You can copy this link, we will need it later, after we create our Slack App.

  1. Create a Slack App

You will have to create a new app in Slack, go to the Slack app directory of your Slack and click on "Create New App". Give the App a name (ex: Meta-API Test) and select your workspace. Now we will configure it.

  1. Configure the Slack App

Open your new App and go to "OAuth & Permissions" and add the URL we copied few minutes ago and "Save URLs". You should have something looking like this:

Now return to "Basic Information" where you'll find a part name App Credentials (see image below). This is where you'll the Client ID and Client Secret we need for Meta-API.

Copy and paste the Client ID and Client Secret in the authentication modal of Meta-API (see image below) and select the permission chat:write:user in "Permission needed". Leave the scope delimiter empty, Slack doesn't need it.

Save the authentication.

danger

Do not "Save and quit" for now as we need to authorize the application.
In case you quit this window, you can return to the "Connectors Authentication" on the menu on the right and edit the previously created Slack auth.

  1. Authorize the application

Now, click on "Authorize" and follow the instructions that appears.

  1. Save and quit

Et voilà! You're done. You have successfully configure the OAuth for Slack. Let's select your new auth for Slack and an OAuth Profile.

Linked Slack to the Spell: send gif and movie's information to a Slack channel​

Everything is set up, now we want to send our movie and gif to a Slack channel.
For this, we will have to prepare our text in a variable and pass it through a variable. To do this, we can use the Slack Block Kit Builder and get the JSON array needed.

To help you, here is an extract of code:

note

For convenience, we extract the movie poster, the vote average and the gif URL. All these info are on the code below.

Paste this code after the Giphy Connector and before the Slack Connector.

const { vote_average, poster_path } = tmdbResult.results[0];

const gifUrl = giphyResult.data.images.downsized_large.url;

// Format poster image to be 500px wide.
const poster = `https://image.tmdb.org/t/p/w500/${poster_path}`;

const slackBlocks = [
    {
        "type": "section",
        "text": {
            "type": "mrkdwn",
            "text": `*${original_title}*\n${vote_average} / 10`
        },
        "accessory": {
            "type": "image",
            "image_url": poster,
            "alt_text": `${original_title} poster`
        }
    },
    {
        "type": "image",
        "title": {
            "type": "plain_text",
            "text": `A nice gif for ${original_title}`,
            "emoji": true
        },
        "image_url": gifUrl,
        "alt_text": "marg"
    }
];

const encodedBlocks = JSON.stringify(slackBlocks);

Now in the Slack Connector's detail panel, we will link the input blocks to encodedBlocks and put in channel a test channel like "meta-api-test", you should create it first. (see image below)

Save and Run your Spell: you should see in Slack the movie and the associated gif.

Deploy your Spell in production​

Everything is working, now we want to deploy our Spell in production. To do that, heads up to the "Access Spell's details" and click on "Manage versions"

There, you can see a development version. We will create our first release to production from this point: Click on the "Create a new version" button and fill the form (ex: title: v1.0 description: First release of the tutorial. Connect TMDB, Giphy and Slack together.)

note

The release will save the current state of your Spell and Connectors and create an image of it. You can edit totally your Spell, it won't affect the previous releases.

A new card appears with a "Promote to production" button. Click on it and the release will be automaticaly deployed to your production environment. When the loader has finished, your Spell is publicly accessible through your default Meta-API key (we'll see how to restrict a Spell in the next step).

Get and test the URL of the Spell​

Always in the same modal, you can see a part named "Production". There you will have the production URL to call your Spell and even test it on your browser directly. If you click on "Manage Snippets and API keys", it will also give you some code snippets to integrate the Spell into your own application.
Open it and look inside, here what you'll get:

If you click on "Click here to test the Spell in a new page" it will open a new window and execute your Spell.
Be careful to have a Spell's key selected.

Restrict access to your Spell​

We've got our Spell running and in production, now we would like to have some restriction to it and put specific access.
In the "Snippets and Urls" tab, click on "Access your API keys". It will redirect you on your client account and you'll see your keys set up for your account.

Add a new key by clicking the "Add new API Key" button, a new card will be added to your keys. Give it a name and notice the section "Restrict this key to these Spells:.
This section will authorized this key for only the Spells selected (note that if there is no Spells selected, the key is available for all of your Spell).
Select the Spell A gif from a movie and save your key. You should get this:

Now this key is only active for the Spell A gif from a movie and if you give that key, no-one can access your others Spells.
Also, you can revoke and a key whenever you want. You can deactivate key, too.
