Technical Overview

Overall view of Meta-API#

Meta-API is a SaaS platform for on demand data sourcing. It helps you to connect easily to different APIs, called Connectors, to normalize the data sourced and to give you a customized endpoint - know as Spell - allowing you to access the data you need and want.

Connectors#

A Connector is the key central of Meta-API. It is an API describe with the Open API Specification (OAS) and will give you all the information of the API directly from the platform. You can find here a better understanding of how it works here (French 🇫🇷 only):

Meta-API Connector Manager

Spell#

It's where magic happens! A Spell is your customize API endpoint, you'll be able to connect Connectors together, enhance the data and normalize it. To create your own unique endpoint, follow the guide!

Create a Spell#

To create a Spell, click on "New blank Spell" on the Spells page in Meta-API:

Button to create a brand new Spell

Fill in the information then click Create:

Pop-up for editing information of the new Spell. Note that just a name is mandatory but we strongly advise you to put a description.

After that, you will be redirect to the Editor of your new spell. Let see what is inside now.

The main page of your Spell. You'll be able to connect data, transform and put it in production from here.

What's inside a Spell#

The interface is separate in two: the left side will concentrate all the information of your Spell, the right one is your code editor.

Code Editor#

Our code editor support Typescript (and Javascript). Based on Monaco Editor (source project of VSCode), you'll have many features accessible, such as IntelliCode and syntax validation to help you.

Example of our integrated code editor

⭐ Code autocompletion on APIs responses ⭐

We implemented our developers best loved feature : Code Autocompletion on API 😍

On each connector you'll use, all the responses inside the code will automatically have a TypeScript type and you'll have autocompletion on response variable right into your code editor.

Example of code autocompletion on Google Maps API

Connectors#

The first panel on the right will be your Connectors.

From this panel, you can add, delete update any Connectors from your Spell.

You can manage all the variables, authentication and documentation form it.

More information on this on Meta-API Connector Manager

Testing#

A test panel will help you to develop and test your Spell right into your web browser.

All the responses will be shown below and our console will catch all console.log, console.error etc. inside your code.

You can also use try / catch and throw to manage errors.

Example of a test request's launch

NPM Dependencies#

Meta-API is based on javascript and NodeJS server, that means you can use pretty much all NPM Dependencies in your Spell.

To do so, search a NPM dependency, click on it and it will be added to your packages.

Edit the information if needed:

  • Give an alias for the dependency (ex : _ for lodash or R for Ramda)
  • Pick a specific version of a package.

You can active advanced NodeJS Apis if needed as fs, path, util, ...

Then click Save and you're all set up!

NPM Dependency panel

Information relative to the Spell#

In the last panel, you'll be able to edit the Spell's information as the name, description.

You can see usage counter and performances metrics, too.

This interface allows you to manage all your connected accounts, update all authenticated methods and add new apps.

You can manage OAuth v2 authentication with you own dev credentials

The configuration is generated from the connector's OAS configuration file
