Managing images and files inside a Spell

You have two ways for managing files and images inside a Spell : Buffer or local files.

Managing with buffer#

With a Buffer, you can store files directly inside the Spell's memory.

Here a snippet to store a remote file inside a Buffer :

const imageInBuffer: Buffer = await axios.get("https://images.unsplash.com/photo-1546776310-eef45dd6d63c?ixid=MXwxMjA3fDB8MHxwaG90by1wYWdlfHx8fGVufDB8fHw%3D&auto=format&fit=crop&w=739&q=80", {
responseType: 'arraybuffer'
});

You can also create a Buffer from a string (utf8) or an object :

const fileInBuffer = new Buffer.from(myObject);

Or a base64 string :

const fileFromBase64 = new Buffer(base64String, 'base64');

Then you can manipulate this variable to output a file (see Generate a file from a Spell) or to link it inside a POST request, etc.

Warning : memory are limited inside a Spell. If you tried to store too large files inside a Buffer, you'll break your Spell's execution.

Read and save temporary files#

You'll need to activate the fs Node's library to access Spell's filesystem.

Then you can read and write files from the local tmp directory like this:

//Write a new file named "test" inside the tmp directory and putting inside the content "Hello World"
fs.writeFileSync('test', "Hello World");
//Loading the file's content inside a Buffer
const fileContentBuffer = fs.readFileSync('test');
//Convertir Buffer to string to use the content
const fileContentAsString = fileContentBuffer.toString();
