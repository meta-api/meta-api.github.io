Spell's code editor interface

When you create your account you can create your first Spell by following the onboarding. You can create as many as you want from the Spell page

On the left side you can see all connectors used for the current spell.

In the center you will be able to test your script, manage node dependencies, releases and change information.

On the right you have control over the configuration of your connectors or access to your console.

Manage your connectors‚Äã

Once you have added your connectors, you can configure it in the right of the screen to set up authentication, parameters and variables.

Your connector is available as a variable in the editor (tmdbResult in this example), do not change it or you will not be able to access it.

Information‚Äã

In this page you can find statistics on your spell usage and manage versions.

You can change at anytime your spell name and description of your spell in Spell settings.

Dependencies‚Äã

You can add as much dependencies as you want from the NPM repository. This is the code equivalent of 

 const axios = require('axios');

An alias is the name for the package inside your code. Call the library by this name.

Code Interface‚Äã

You can code in Javascript or in TypeScript (recommended). Your spell can end with a return, usually an object that will be transformed to JSON when called with an HTTP request.

A full autocomplete is available in the code interface for javascript methods, your connectors responses and node dependencies.
