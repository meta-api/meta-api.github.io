Authenticate your GitHub account

In order to use the GitHub API, you'll need to authenticate with an oAuth App

1. Create a new authentication in Meta APIâ€‹

Log in on Meta API dashboard and go to the API's Authentication page. Click on the Create a new authentication button and select the GitHub API. Generate your personnal link and copy it, and then click on Save.

2. Create a GitHub Appâ€‹

Register new GitHub App on your GitHub account settings.

Choose a name, fill your homepage URL (if you don't have one, it will work with anything) and the Callback URL you just copied.

User-to-server token are a beta feature on GitHub and might not work correctly

Make sure to unckeck the Expire user authorization tokens checkbox when you configure your app or Opt-out from the Optional features

Unckeck the Webhook activation

Select the permissions that you'll need in your Spells (you can update your App later to add more). For exemple: request Read & write access for Contents to view and update files

Finnaly, choose if only your private account will use this app or if you plan to use it with an organization account or other GitHub users.

3. Update the Meta API authenticationâ€‹

Copy the Client ID and the Client secret (you'll have to generate one) and past them on the authentication modal in Meta API.

Select the permissions you need and set the scope delimiter with just a space (" ").

Install the App in your GitHub account (or organization) and select the repositories you need to access

4. Authorize your accountâ€‹

Click on the Authorize button, the GitHub popup will open and ask you to authorize it.

Reload the page to see your OAuth profile.
