Manage your API keys

Manage your API keys#

You can directly manage your API keys, needed to launch your Spells, from your client account.

You can attribute labels on your keys and consult the last it was used.

By default, an API key can be use by all your Spells but you can limit this by adding a Spell onto the authorize list of Spells. Simply choose a name of the Spell, save and voilÃ ! The API key will be only active for the selected Spells.

Use of your keys#

When you send a request to Meta API, you have to communicate on of your API key. You have two possibilities for doing so:

Either you put it into the headers of the request as apikey (recommended)

Example :

curl --location --request GET 'https://api.meta-api.io/api/spells/5e21bd14f6f042ce250c5be8/runSync' \
--header 'Content-Type: application/json' \
--header 'apikey: YOUR_API_KEY'

Either you add it into the URL parameters of your request under the name apikey

Example : GET https://api.meta-api.io/api/spells/5e21bd14f6f042ce250c5be8/runSync?apikey=YOUR_API_KEY

Advices#

We recommend you to change your API keys regularly. For doing so, you can follow these steps:

  1. Create a new API key and add any limitation of Spell you need;

  2. Change the references into your application code;

  3. Wait a few hours until the last use of your key is acceptable (Â± 1 day);

  4. Deactivate the older key (do not delete it right away). Deactivate a key allows you to ensure that the production isn't impacted and if it is, you can rollback easily as a key is unique ;

  5. If everything is good, you can delete the key.
