Working with connector value

Anatomy of the connector's code​

This is an example of a connector's code snippet when you add a connector to your Spell:

//#region Start of Airtable connector
const airtableResult: Datagetbaseidbasename = await connectorService.config(
  "62b7363ed54c127d638c4d70",
  {}
);
console.info({ Airtable: airtableResult });
//#endregion End of Airtable connector

We'll decrypt all part of this code snippet:

  • const airtableResult: the variable declaration where the API's response will be stored.
  • : Datagetbaseidbasename: this is the Typescript notation to add type to the variable. That will give you autocompletion of the API's response schema inside your code.
  • await connectorService.config(: our internal connector's service to link the spell's configuration to your code.
  • "62b7363ed54c127d638c4d70": the unique configuration ID of your added connector. This ID is calculated when you add the connector to your Spell.
  • {}: this empty object is the body when you call a POST endpoint for example. For a GET request, this object is unused.
  • console.info({ "Airtable" : airtableResult });: we add a console.info for you to have the API's response inside your logs.

Finally, this snippet is surrounded by comments (//) to make them more remarkable inside the code, you can even collapse the connector's code to give you some space.

Adapt the code to your needs​

You can adjust this snippet of code to your need:

  • Change the variable name (airtableResult in our example) to use a more relevant name (myUsers for example)
  • You can remove the Typescript annotation (: Datagetbaseidbasename in our example) if it's creating conflicts.
  • Surround this snippet of code with try / catch or libraries like bottleneck or async-retry to add mechanisms to your connector.

Get data result from a connector​

Here are two ways to retrieve data from your connectors

Destructuring data​

You can use JS destructuring assignment to extract sub properties from your connector's variable.

For example: if you have a connector that'll return data like this:

{
    result: {
        id: "abc123",
    }
}

you can use destructuring assignment to write your connector's code like this:

//#region Start of Airtable connector
const {
  result: { id },
}: Datagetbaseidbasename = await connectorService.config(
  "62b7363ed54c127d638c4d70",
  {}
);
console.info({ ID: id });
//#endregion End of Airtable connector

That will create a id variable with the content of the id sub property inside.

Accessing sub property with dot notation​

The classical way to access data is to use the JS dot notation.

For example, if you have this result from a connector:

{
    result: {
        id: "abc123",
    }
}

You can access the id property like this:

//#region Start of Airtable connector
const airtableResult: Datagetbaseidbasename = await connectorService.config(
  "62b7363ed54c127d638c4d70",
  {}
);
console.info({ Id: airtableResult.result.id });
//#endregion End of Airtable connector

Our code editor embraces fully JavaScript and TypeScript, so you can do the most of every API you use and in one of the common language.
